# Конспект: Функциональное программирование

## 1. Pure Functions (Чистые функции)

**Определение.** Чистая функция — это функция со следующими свойствами:

1. **Отсутствие побочных эффектов (no side effects)** — выполнение функции не изменяет состояние программы вне себя:
   - не меняет глобальные переменные;
   - не использует потоки ввода-вывода (I/O).

2. **Детерминированность (referential transparency)** — для одних и тех же аргументов функция всегда возвращает одно и то же значение.

**Следствия:**
- Результат чистой функции зависит только от её входных аргументов.
- Чистая функция не зависит от глобального состояния или «внешнего мира».
- Концептуально чистая функция похожа на математическую функцию: для любого входа возможен ровно один выход.
- Разным входам могут соответствовать одинаковые выходы (например, `abs(-5)` и `abs(5)` дают `5`).

**Примеры (Python, C++, C#):**

```python
# Python — Чистая функция
def add(a, b):
    return a + b

# Python — НЕ чистая (побочный эффект)
total = 0
def add_to_total(x):
    global total
    total += x
    return total
```

```cpp
// C++ — Чистая функция
int add(int a, int b) {
    return a + b;
}

// C++ — НЕ чистая (меняет глобальное состояние)
int total = 0;
int add_to_total(int x) {
    total += x;  // побочный эффект
    return total;
}
```

```csharp
// C# — Чистая функция
int Add(int a, int b) => a + b;

// C# — НЕ чистая (запись в статическое поле)
static int total = 0;
int AddToTotal(int x) {
    total += x;  // побочный эффект
    return total;
}
```

## 2. Generators (Генераторы)

**Определение.** Генераторная функция — это функция, которая ведёт себя как итератор, т.е. её можно использовать в `for`-цикле. Python предоставляет поддержку generator functions и generator expressions, что упрощает написание кода.

---

### Упрощение кода (Simplified Code)

Сравним три способа реализовать `firstn(n)` — первые *n* неотрицательных целых чисел (пусть *n* очень большой, и каждое число условно занимает много памяти).

**1. Построение списка и возврат — всё в памяти:**

```python
def first_n(n):
    '''Build and return a list'''
    num, nums = 0, []
    while num < n:
        nums.append(num)
        num += 1
    return nums

sum_of_first_n = sum(first_n(1000000))
```

Неприемлемо: полный список держится в памяти.

**2. Паттерн итератора (класс) — много boilerplate:**

```python
class first_n:
    def __init__(self, n):
        self.n = n
        self.num = 0

    def __iter__(self):
        return self

    def __next__(self):
        if self.num < self.n:
            cur, self.num = self.num, self.num + 1
            return cur
        raise StopIteration()

sum_of_first_n = sum(first_n(1000000))
```

Работает, но много шаблонного кода и «запутанная» логика.

**3. Генераторная функция — коротко и естественно:**

```python
def firstn(n):
    num = 0
    while num < n:
        yield num
        num += 1

sum_of_first_n = sum(firstn(1000000))
```

Та же логика, что у версии со списком, но с характеристиками итератора по памяти.

*Примечание: для такого случая в Python 3 удобнее `range(n)` — он уже возвращает sequence-type, а не список.*

---

**Generator expression** — ещё один shortcut. List comprehension можно превратить в генератор, заменив `[]` на `()`:

```python
# list comprehension — создаёт список в памяти
doubles = [2 * n for n in range(50)]

# generator expression — лениво; эквивалент list(...)
doubles = list(2 * n for n in range(50))
```

Или думать о list comprehension как о generator expression, обёрнутом в `list(...)`.

Генераторы — особый тип итератора; `list`, `set` и т.д. — итерируемые объекты. Единообразие обработки упрощает код.

---

### Производительность (Improved Performance)

Выигрыш от генераторов — **ленивая (on demand)** генерация: меньше памяти, не нужно ждать полной генерации всех элементов, чтобы начать их использовать.

**Ограничение:** генератор даёт выигрыш, только если значения **не нужны более одного раза**.

```python
# Плохо: генерация дважды (product — любая функция, потребляющая итератор)
s = sum(range(1000000))
p = product(range(1000000))

# Лучше: один раз в список, два прохода
nums = list(range(1000000))
s = sum(nums)
p = product(nums)
```

Если хранить все объекты в памяти невозможно, генератор остаётся единственным вариантом, даже при дублировании вычислений.

В Python 3 `range()` уже возвращает sequence-type вместо списка — как бы ленивый объект.

---

### Примеры (Examples)

**Композиция генераторов:**

```python
# генератор квадратов
square = (i * i for i in range(1000000))
total = 0
for i in square:
    total += i
```

**Композиция с `takewhile` — квадраты меньше 100:**

```python
from itertools import count, takewhile

square = (i * i for i in count())
bounded_squares = takewhile(lambda x: x < 100, square)
total = sum(bounded_squares)
```

**Одна операция — list vs generator:**

```python
# list comprehension
def double(L):
    return [x * 2 for x in L]
eggs = double([1, 2, 3, 4, 5])

# generator function
def double(L):
    for x in L:
        yield x * 2
eggs = double([1, 2, 3, 4, 5])  # eggs — генератор

# то же через generator expression
eggs = (x * 2 for x in [1, 2, 3, 4, 5])

# если нужен список
eggs = list(double([1, 2, 3, 4, 5]))  # или eggs = [x * 2 for x in L]
```

**Сложная логика с состоянием** — генераторная функция лучше, чем expression:

```python
def unique(iterable, key=lambda x: x):
    seen = set()
    for elem, ekey in ((e, key(e)) for e in iterable):
        if ekey not in seen:
            yield elem
            seen.add(ekey)
```

Временное хранилище `seen` инкапсулировано; в expression это было бы громоздко.

## 3. Closures (Замыкания)

**Определение.** Замыкание в Python — это «функция с памятью». Оно позволяет функции сохранять доступ к значениям из окружения, в котором она была создана, даже после завершения работы этого окружения. Используется в функциональном программировании, event handling и callback'ах, когда нужно сохранять состояние без глобальных переменных.

**Как создаётся замыкание:**
1. Функция определена внутри другой функции (вложенная функция).
2. **Внутренняя функция** ссылается на переменные **внешней функции**.
3. **Внешняя функция** возвращает **внутреннюю функцию**.

---

### Примеры

**Пример 1 — фиксация аргумента:**

```python
def outer_function(x):
    def inner_function(y):
        return x + y  # 'x' запоминается из outer_function
    return inner_function

closure = outer_function(10)
print(closure(5))   # 15
print(closure(20))  # 30
```

`outer_function(10)` возвращает `inner_function` с зафиксированным `x = 10`. Возвращённая функция по‑прежнему имеет доступ к `x`, хотя `outer_function` уже завершилась.

**Пример 2 — счётчик с состоянием:**

```python
def make_counter():
    count = 0
    def counter():
        nonlocal count
        count += 1
        return count
    return counter

counter1 = make_counter()
print(counter1())  # 1
print(counter1())  # 2
```

**Пример 3 — фабрика функций (multiplier):**

```python
def make_multiplier(factor):
    def multiply_by(x):
        return x * factor
    return multiply_by

double = make_multiplier(2)
print(double(5))  # 10

# Проверка «запомненных» переменных
print(double.__closure__[0].cell_contents)  # 2
```

---

### Внутреннее устройство

При создании замыкания Python сохраняет переменные внешней функции в атрибуте `__closure__`. Внутренняя функция хранит **ссылку**, а не копию этих переменных.

---

### Типичные применения

- **Function Factories** — функции, порождающие другие функции с заданным поведением.
- **Data Hiding** — сокрытие переменных от прямого доступа.
- **Event Handling** — сохранение состояния между взаимодействиями с пользователем.
- **Callbacks** — передача функций с сохранением контекста.

## 4. Decorator (Декоратор)

**Определение.** Decorator (также Wrapper) — структурный паттерн проектирования, который позволяет динамически добавлять объектам новое поведение, помещая их в специальные wrapper-объекты с этим поведением.

**Проблема:** наследование статично и не даёт комбинировать поведения (например, уведомления: email + SMS + Slack — комбинации ведут к комбинаторному взрыву подклассов).

**Решение:** использовать **композицию** вместо наследования. Wrapper реализует тот же интерфейс, что и обёрнутый объект, делегирует ему запросы, но может изменить результат до или после вызова.

---

### Структура (Refactoring.Guru)

1. **Component** — общий интерфейс для wrappers и обёрнутых объектов.
2. **Concrete Component** — базовое поведение.
3. **Base Decorator** — хранит ссылку на обёрнутый объект, делегирует ему операции.
4. **Concrete Decorators** — добавляют своё поведение до/после вызова обёрнутого объекта.
5. **Client** — оборачивает компоненты в несколько слоёв декораторов.

---

### Пример (OOP)

```python
class Component:
    def operation(self) -> str:
        pass

class ConcreteComponent(Component):
    def operation(self) -> str:
        return "ConcreteComponent"

class Decorator(Component):
    def __init__(self, component: Component):
        self._component = component

    def operation(self) -> str:
        return self._component.operation()

class ConcreteDecoratorA(Decorator):
    def operation(self) -> str:
        return f"DecoratorA({self._component.operation()})"

class ConcreteDecoratorB(Decorator):
    def operation(self) -> str:
        return f"DecoratorB({self._component.operation()})"

# Client: стек декораторов
simple = ConcreteComponent()
decorated = ConcreteDecoratorB(ConcreteDecoratorA(simple))
print(decorated.operation())  # DecoratorB(DecoratorA(ConcreteComponent))
```

---

### Python: декораторы функций

В Python синтаксис `@decorator` — частный случай паттерна: функция-обёртка применяется к другой функции (часто через closure):

```python
def bold(func):
    def wrapper(*args, **kwargs):
        return f"<b>{func(*args, **kwargs)}</b>"
    return wrapper

@bold
def greet(name):
    return f"Hello, {name}"

print(greet("World"))  # <b>Hello, World</b>
```

Тот же код без `@`: `greet = bold(greet)`.

---

### Плюсы и минусы

**Плюсы:** расширение поведения без новых подклассов; добавление/снятие ответственностей в runtime; комбинирование нескольких обёрток.

**Минусы:** сложно удалить отдельный wrapper из стека; порядок декораторов влияет на результат.

## 5. First-class Functions (Функции как объекты первого класса)

**Определение.** Язык поддерживает **First-class functions**, когда функции обрабатываются как обычные переменные: их можно передавать в другие функции как аргументы, возвращать из функций и присваивать переменным.

---

### Примеры (Python)

**Присвоение функции переменной:**

```python
def say_foobar():
    print("foobar")

foo = say_foobar
foo()  # Вызов через переменную → foobar
```

**Передача функции как аргумента:**

```python
def say_hello():
    return "Hello, "

def greeting(hello_func, name):
    return hello_func() + name

print(greeting(say_hello, "Python!"))  # Hello, Python!
```

Функция, передаваемая как аргумент, называется **callback-функцией**.

**Возврат функции из функции:**

```python
def make_greeter():
    def greet():
        print("Hello!")
    return greet

greeter = make_greeter()
greeter()  # Hello!
```

Функция, которая возвращает другую функцию или принимает функции как аргументы, называется **higher-order function** (функция высшего порядка).

---

### Связь с другими темами

- **Closures** — внутренняя функция «запоминает» окружение внешней; часто возвращается как first-class объект.
- **Decorator** — паттерн опирается на передачу и возврат функций.
- **map, filter, sorted(key=...)** — принимают функции как аргументы.
